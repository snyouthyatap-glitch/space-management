# 일지 양식 정리

이 문서는 웹사이트에서 받는 입력값과 Apps Script가 그대로 저장하는 값의 기준표입니다.

운영 기준으로는 이 문서를 두는 것이 도움이 됩니다.

- 어떤 값을 꼭 받아야 하는지 바로 확인할 수 있음
- 관리자 대리 제출과 수동 제출도 같은 구조로 맞출 수 있음
- 추후 스프레드시트, 캘린더, 네이버 예약 연동 시 필드가 흔들리지 않음

## 1. 라운지 방문 일지

### 화면 입력
- 성별
- 생년월일

### 내부 처리
- 나이 자동 계산
- 1일 임시회원 생성
- 방문일지 자동 제출

### 저장 기준
- 날짜
- 성별
- 나이
- 방문 구분: `lounge_guest`
- 생성 방식: `lounge`

## 2. 시설 회원 가입

### 화면 입력
- 성함
- 성별
- 나이
- 연락처

### 내부 처리
- 전화번호 숫자만 남김
- 끝자리 4자리 생성
- 기존 전화번호 중복 확인
- 신규 회원이면 즉시 등록
- 등록 직후 방문일지 자동 제출

### 저장 기준
- 성함
- 성별
- 나이
- 연락처
- 전화번호 끝자리 4자리
- 상태: `approved`
- 역할: `user`

## 3. 방문 일지

### 자동 제출되는 경우
- 기존 시설 회원이 QR 진입 성공
- 신규 시설 회원 가입 직후
- 라운지 이용 입력 완료 직후

### 관리자 수동/대리 제출 가능
- 방문 누락 시 대리 제출
- 필요 시 수동 제출

### 시트 최종 양식
- 연번
- 날짜
- 성별
- 나이
- 사용 매수

## 4. 프린터 일지

### 화면 입력
- 사용 매수

### 제약
- 1일 최대 10장

### 시트 최종 양식
- 연번
- 날짜
- 성별
- 나이

## 5. 커리어존 일지

### 화면 입력
- 시작 시간
- 종료 시간
- 이용 목적
- 추가 인원 수
- 추가 인원별 성별
- 추가 인원별 나이

### 시트 최종 양식
- 연번
- 날짜
- 사용목적
- 인원수
- 각 인원의 나이와 성별
- 시작시간
- 종료시간
- 20~29세(남)
- 20~29세(여)
- 30~39세(남)
- 30~39세(여)
- ~19세(남)
- ~19세(여)

## 6. 커넥트존 일지

### 화면 입력
- 시작 시간
- 종료 시간
- 이용 목적
- 추가 인원 수
- 추가 인원별 성별
- 추가 인원별 나이

### 시트 최종 양식
- 연번
- 날짜
- 사용목적
- 인원수
- 각 인원의 나이와 성별
- 시작시간
- 종료시간
- 20~29세(남)
- 20~29세(여)
- 30~39세(남)
- 30~39세(여)
- ~19세(남)
- ~19세(여)

## 7. 관리자 검색/대리 제출

### 검색 기준
- 성함
- 전화번호 일부
- 끝자리 4자리

### 대리 제출 가능 항목
- 방문
- 프린터
- 커리어존
- 커넥트존

## 8. 수동 제출

### 화면 입력
- 성함
- 성별
- 나이
- 연락처(선택)
- 날짜
- 일지 종류
- 종류별 추가 항목

### 특징
- 회원이 아니어도 기록 가능
- 운영자가 누락 건을 직접 보완할 때 사용

## 9. 권장 운영 방식

이 구조에서는 **양식을 따로 고정해두는 것이 좋습니다.**

특히 아래 세 가지에 도움이 됩니다.

1. 현장 안내문 제작
2. 관리자 교육
3. Apps Script와 시트 구조 유지

즉, 화면은 단순하게 두되, 뒤에서는 이 문서 기준으로 필드를 고정하는 방식이 가장 안정적입니다.
